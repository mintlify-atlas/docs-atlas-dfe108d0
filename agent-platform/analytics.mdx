|---|---|
| **Users** | Unique active users in the period |
| **Sessions** | Total conversation sessions |
| **Messages** | Messages exchanged (user + agent) |
| **Tokens** | Total token consumption |

## Sessions

Sessions track continuous user interactions, grouping all traces from a single conversation.

Each session shows: user ID, start time, duration, trace count, agents used, and total cost.

## Audit logs

Track all changes made across your account.

**What's logged:**
- User actions (create, update, delete)
- Configuration changes
- Deployments
- Access events

Each log entry includes event name, category, user name, date and time, and a detailed description.

## Best practices

- Monitor **success rate**, **latency**, **cost**, and **user satisfaction** as key metrics.
- Set baselines for normal ranges to detect anomalies.
- Review daily for health checks, weekly for trend analysis, and monthly for deep dives.
- Use insights to drive improvements: slow agent? Optimize tools or prompts. High error rate? Review configurations.
