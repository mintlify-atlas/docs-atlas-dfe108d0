App deployment is the process of moving an application version from development to live environments in a controlled and repeatable manner. It ensures that tested and stable versions of the app are accessible to users in the appropriate environments.

## Deployment lifecycle

<Steps>
  <Step title="Create an application version">
    Build and package the app into a deployable, versioned snapshot.
  </Step>
  <Step title="Create an environment">
    Isolate configurations to ensure updates and tests don't impact end users.
  </Step>
  <Step title="Deploy the app">
    Release a selected version to a chosen environment.
  </Step>
  <Step title="Access the deployed app">
    Via a unique URL or API for integration or user interaction.
  </Step>
</Steps>

<Note>Only authorized users can access version control and environment management features.</Note>

## App versioning

The app versioning system manages the entire app lifecycle—including associated AI agents and tools—ensuring version consistency, stable releases, and streamlined deployment.

### Key benefits

- **Immutable releases**: Once deployed, versions remain unchanged, guaranteeing stability.
- **Controlled deployments**: Implement updates smoothly without disrupting services.
- **Dependency protection**: Tools linked to active versions can't be deleted.
- **Environment isolation**: Deploy different app versions to separate environments.
- **Rollback support**: Revert to previous versions while retaining complete version history.

### Create an app version

1. Go to the **Versions** page and click **+ New Version**.
2. Enter the **Name** and **Description**. Select the appropriate versions of agents and tools to include.
3. Click **Create**.
4. Preview the version to review its configuration, then **Deploy** it.

### Deploy an app version

Select the app version and environment from the dropdown, then click **Proceed**.

### Restore a version

Restoring a version reverts the current draft to a previously saved state. This action **overwrites the entire draft**—including all configurations, tools, and agent settings.

<Warning>Restoring a version replaces the current draft and cannot be undone. Consider saving your current work as a new version before restoring another.</Warning>

### Delete a version

Before deletion, the platform performs dependency checks to ensure no linked agents or tools are impacted.

## Environments

Environments are isolated deployment targets with their own configurations. The platform supports multiple environments; by default, a draft environment exists for development and testing.

### Key benefits

- **Dedicated endpoints**: Each environment provides a unique URL for testing or integration.
- **Deployment history**: View detailed logs of all deployments across environments.
- **Environment isolation**: Each environment maintains separate URL endpoints, independent configuration, its own deployment history, and isolated API keys.

### Create an environment

1. Go to the **Environments** page.
2. Click **+ New Environment**.
3. Enter the **Name**, **Description**, and select the **App Version**.
4. Click **Create**.

### Manage an environment

- **Linked app version**: To switch app versions, click **Change**, select the desired version, and click **Deploy**.
- **Web client details**: View the URL generated by the platform.
- **cURL script**: Use the curl script to interact with the app through code.
- **Delete environment**: Removes the environment with user confirmation, deactivates endpoints, and cleans up configurations.

## Access methods

### Web widget (webclient script)

Embed the agentic app within your existing website using a single `<script>` tag.

```html
<!-- Add to your website -->
<div id="chat-container"></div>
<script src="https://widget.kore.ai/v1/loader.js"
        data-app-id="your-app-id"
        data-container="chat-container">
</script>
```

```javascript
KoreWidget.init({
  appId: 'your-app-id',
  containerId: 'chat-container',
  theme: 'light',  // or 'dark'
  position: 'bottom-right',
  greeting: 'Hi! How can I help you today?'
});
```

### API integration

```bash
curl -X POST https://api.kore.ai/v1/apps/{app_id}/execute \
  -H "Authorization: Bearer {api_key}" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What is your return policy?",
    "session_id": "user_session_123",
    "metadata": {
      "user_id": "user_456"
    }
  }'
```

**Response:**
```json
{
  "response": "Our return policy allows returns within 30 days...",
  "session_id": "user_session_123",
  "agent": "support_agent",
  "tools_used": ["knowledge_search"],
  "tokens": {
    "input": 45,
    "output": 128
  }
}
```

## API keys

API keys are required to authenticate requests sent to the agentic app via APIs.

### Generating keys

1. Navigate to **Deploy** → **API Keys**.
2. Click **Generate Key**.
3. Enter a descriptive name and click **Generate Key**.
4. Copy and store securely.

<Warning>For security reasons, the key cannot be viewed again once you navigate away from the page.</Warning>

## Rollback

1. Navigate to **Deploy** → **History**.
2. Find the previous working version.
3. Click **Rollback**.
4. Confirm the action.
