title: "Inbox"

---

The Inbox provides a centralized queue for human tasks generated by workflows:

- Review AI-generated outputs.
- Approve or reject workflow decisions.
- Provide missing information.
- Handle exceptions and escalations.

## How it works

When a workflow reaches a Human Node, execution pauses and a task is routed to the Inbox. The assigned reviewer completes the task — approving, editing, or rejecting — and the workflow resumes from the next node.

For details on configuring the Human Node, see [Human node](/ai-for-process/nodes#human-node).

## Inbox interface

### Task list

| Column | Description |
|---|---|
| **Task** | Task name and description. |
| **Workflow** | Source workflow. |
| **Priority** | High, Medium, or Low. |
| **Due** | Deadline for completion. |
| **Assigned** | Current assignee. |
| **Status** | Pending, In Progress, or Completed. |

### Filters

Filter tasks by workflow name, priority level, due date range, assignee, status, or tags.

### Search

Search tasks by task content, workflow ID, custom fields, or notes.

## Task types

### Review task

Review and optionally edit AI-generated content. Typical actions: approve, edit and approve, or reject.

### Approval task

Binary decision (approve or reject) with optional comments. You can require a comment on rejection.

### Input task

Collect information from a human reviewer when the workflow lacks required data to continue. Fields are presented as forms with text, dropdown, address, and other input types.

### Classification task

Categorize or label content for downstream processing. Reviewers select from predefined categories and priorities.

## Assignment

### Assignment rules

Configure how tasks are assigned:

- Assign by role (for example, assign approval tasks to the `approver` role).
- Assign based on workflow data (for example, route high-value items to `senior_approver`).
- Use round-robin assignment to distribute tasks evenly across a team.

### Manual assignment

To reassign a task:

1. Open the task details.
2. Click **Reassign**.
3. Select the new assignee and add an optional note.
4. Confirm the reassignment.

### Self-assignment

Enable self-assignment to allow users to claim tasks from the queue. Configure the maximum number of concurrent tasks per user.

## SLAs and escalation

### SLA configuration

Set time-based expectations for task completion:

- **Warning threshold**: Notify the assignee when approaching the deadline.
- **Critical threshold**: Notify the supervisor when near a breach.
- **Breach threshold**: Escalate when the deadline passes.

### Escalation rules

Automatic escalation when deadlines approach or are breached:

- **Level 1** (warning): Notify the assignee and highlight the task.
- **Level 2** (critical): Notify both assignee and supervisor; optionally reassign to the supervisor.
- **Level 3** (breach): Notify the full chain and reassign to a manager with high priority.

## Bulk actions

Process multiple tasks at once:

| Action | Description |
|---|---|
| **Bulk approve** | Approve multiple selected tasks. |
| **Bulk assign** | Assign selected tasks to a user. |
| **Bulk prioritize** | Update priority for selected tasks. |
| **Bulk tag** | Add labels to selected tasks. |

A confirmation prompt is required before bulk actions execute. Maximum batch size is 50 tasks.

## Best practices

- Keep tasks focused with clear instructions and sufficient context.
- Provide original data for reference alongside AI-generated output.
- Set realistic deadlines and balance workload across the team.
- Match task types to reviewer skills.
- Use escalation rules to prevent tasks from blocking workflows.
