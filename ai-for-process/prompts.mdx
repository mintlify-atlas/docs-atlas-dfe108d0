Prompt Studio manages the full lifecycle of prompts — from creation and testing to optimization and deployment. You can test prompts across external, fine-tuned, and open-source models to find the best model and configuration through iteration.

## Key features

- 65+ pre-built templates for common use cases.
- Multi-model comparison and testing across up to 5 models simultaneously.
- Variable support to test multiple scenarios at once.
- AI-assisted prompt generation and synthetic test data creation.
- Version control, draft history, and collaboration.
- Model performance analytics: response time and token usage.

## Access Prompt Studio

<Steps>
  <Step title="Open Prompts">
    Log in to your AI for Process account and click **Prompts** in the top navigation bar.
  </Step>
  <Step title="Select a tab">
    On the **Prompts** dashboard, select:
    - **All prompts**: All available prompts.
    - **My prompts**: Prompts you created or saved.
    - **Shared prompts**: Prompts shared with you.
  </Step>
  <Step title="Create a prompt">
    Click **New prompt**, enter a name, and click **Proceed**. The Prompt landing page opens.
  </Step>
</Steps>

## Create a prompt

From the Prompt landing page, choose one of three ways to start:

### Generate a prompt

Expands a short input (one or two sentences) into a detailed, structured prompt.

<Note>Only OpenAI and Anthropic models are supported for prompt generation and test data generation.</Note>

<Steps>
  <Step title="Open the generator">
    Click **Generate a prompt**.
  </Step>
  <Step title="Configure and generate">
    In the Prompt generator dialog, select a model and enter your instruction.
  </Step>
  <Step title="Apply">
    Review the AI-generated prompt and click **Proceed** to copy it to the canvas. Customize as needed.
  </Step>
</Steps>

### Start from scratch

Opens a blank prompt canvas where you add prompts, variables, and models. You can also pull in templates from the prompt library.

### Prompt Library

Includes 65+ built-in templates for common use cases such as code generation, summarization, content creation, and Q&A.

<Note>All templates are read-only. You can edit content only after importing a template to the prompt canvas.</Note>

<Steps>
  <Step title="Open the library">
    Click **Prompt library** on the landing page.
  </Step>
  <Step title="Select a template">
    Browse the **All templates** or **My templates** tab. Click a template to preview it, then click **Use template**.
  </Step>
  <Step title="Customize">
    The template loads on the canvas. Edit it as needed.
  </Step>
</Steps>

## Work on the prompt canvas

The prompt canvas is the core workspace for testing and comparing AI model performance. The workflow follows four steps:

```text
Add prompts → Apply variables → Select models → Generate output
```

### Add prompts

1. In the **System prompt** field, assign a role to the model (optional).
2. In the **Prompt** field, enter your instructions. Click **Generate Prompt** to expand a short instruction into a more detailed prompt.
3. Optionally define a **Response JSON schema** to structure model responses.

<Note>
- **System prompts** guide the model's overall behavior or tone.
- **Human prompts** specify what the user wants.
</Note>

### Apply variables

Variables let you run prompts with multiple values simultaneously. Use the `{{variable}}` syntax anywhere in the prompt.

1. Add variables in double curly braces in the **Prompt** field. For example, `{{customer_name}}`.
2. In the **Variables** column, assign a value to each variable. Click **Add an empty row** to add multiple rows.
3. Verify that variables are substituted correctly in the Prompt window.

For datasets, map CSV column names to variable names (case-sensitive). For example, `{{Name}}` maps to a CSV column named `Name`.

### Select models

Test prompts with up to 5 models simultaneously:

1. Click the **Select Model** field and choose a model and connection.
2. To add more models, select from the columns on the right.
3. Click the model settings icon to adjust parameters: temperature, top k, top p, and max tokens.

### Generate output

Click **Run** to generate output.

<Note>You can generate a maximum of 10 rows of data simultaneously.</Note>

The output area displays model responses, total input and output tokens, and time taken to generate each response.

## Advanced features

### Bookmark a model

Bookmarking sets a model as your preferred model for a prompt. The bookmarked model is recorded when you commit a version.

1. Click the **Bookmark model with its settings** icon on the model column.
2. If you commit without bookmarking, a dialog prompts you to select a preferred model.
3. After committing, click **Versions** to see the preferred model in version history.

### Draft history

Draft history saves the complete state of the prompt canvas at different points in time, including inputs, outputs, model selections, and variable values. Click **Draft History** from the three dots menu to open the dialog and restore a previous state.

<Note>
- **Draft History** captures both inputs and outputs — the full context of each iteration.
- **Versions** track only prompt inputs (system and human prompts), not outputs.
</Note>

### Manage prompt versions

Prompt versioning tracks iterations in a shared repository. Each committed version records the prompt, system prompt, and preferred model.

**Key behaviors:**
- You must generate output before committing. The first commit creates V1; subsequent commits create V2, V3, and so on.
- The latest committed version is the default. To change it, select a version and click **Mark as a default version**.
- Load any version as a draft to edit without affecting the original. Commit the draft to create a new version.

**To commit a version:**

<Steps>
  <Step title="Commit">
    Click **Commit** on the canvas.
  </Step>
  <Step title="View history">
    Click **Versions** to view the version history.
  </Step>
  <Step title="Use as draft">
    Select a version and click **Use as a draft** to load it on the canvas for editing.
  </Step>
</Steps>

### Share prompts

Prompt sharing lets you collaborate by sharing prompts — including inputs, outputs, and settings — with other users.

| Role | Permissions |
|---|---|
| **Full (Owner)** | View, edit, restore, commit, and delete. Manage users, API keys, and test data. |
| **Edit (Collaborator)** | All Full permissions except delete. |
| **View (Viewer)** | View prompts and versions only. Cannot edit, delete, or commit. |

### Prompt API endpoint

The Prompt API lets you access prompts externally using version-specific API keys. The endpoint is generated automatically when you commit the first version.

- Each prompt has one API endpoint. By default, it returns the latest version.
- A successful request returns the `SystemPrompt` and `HumanPrompt` from the specified version.
- You can create multiple API keys per endpoint. Each key can be copied once and deleted but cannot be reused.

Supported request formats: cURL, Python, Node.js.

## Best practices

- **Start with templates**: Use the prompt library for common use cases to accelerate creation.
- **Use variables**: Run multiple prompts simultaneously for consistent testing across scenarios.
- **Bookmark strong models**: Track better-performing models by bookmarking them.
- **Use version control**: Commit versions for significant changes.
- **Export results as CSV**: Save important outputs for sharing and analysis.
- **Save successful prompts**: Store effective prompts as templates in the library for future use.
