---
title: "Audit logs"
description: "Track and audit all user actions and system events in AI for Process, including logins, role changes, model updates, and workflow activities."
---

**Audit Logs** on the Settings console provides full visibility into user actions and system interactions across your AI for Process account. Every log entry includes:

- Event name and category.
- The user who performed the action.
- Date and time of the event.
- A detailed description of the action.

Use audit logs to identify patterns in user activity, detect anomalies, spot unauthorized usage, and ensure compliance with internal policies and regulations.

## Access Audit Logs

<Steps>
  <Step title="Open Settings">
    Go to **Settings** in the AI for Process top menu.
  </Step>
  <Step title="Navigate to Audit Logs">
    In the left menu, click **Monitoring** > **Audit Logs**.
  </Step>
</Steps>

## Dashboard information

| Column | Description |
|---|---|
| **Event Name** | The specific event or action that occurred. |
| **Category** | The module or entity affected by the event. |
| **User Name** | The user who performed the action. |
| **Date and Time** | When the event occurred. |
| **Description** | Detailed information about what was done. |

## Filter Audit Logs

### Add a custom filter

<Steps>
  <Step title="Open the filter">
    Go to **Audit Logs**. Click the **Filter** icon and click **+ Add Filter**.
  </Step>
  <Step title="Configure">
    Select the **Column**, **Operator** (Is Equal To or Is Not Equal To), and **Value**. Click **Apply**.
  </Step>
</Steps>

To clear filters, click **Clear All**.

### Add multiple filters

Combine multiple filters using **AND** or **OR** operators.

<Note>AND and OR operators cannot be mixed in the same filter set. Use either AND or OR consistently across all filter criteria.</Note>

- **AND**: All conditions must be met for an entry to appear.
- **OR**: Any condition being met is sufficient.

To add multiple filters, select the **AND** or **OR** operator tab in the **Filter By** window before configuring each filter.

## Time-based Audit Logs

Use the time selection feature to view logs within a specific period. Default is **Last 30 Days**.

| Option | Description |
|---|---|
| **All Time** | All logs since the account was created. |
| **Today** | Logs generated on the current day. |
| **Yesterday** | Logs generated on the previous day. |
| **This Week** | Logs for all days in the current week. |
| **This Month** | Logs for all days in the current month. |
| **Last Month** | Logs for all days in the previous month. |
| **Last 30 Days** | Logs for the past 30 days. |
| **Last 90 Days** | Logs for the past 90 days. |
| **This Year** | Logs for all days in the current year. |
| **Last Year** | Logs for all days in the previous year. |

<Steps>
  <Step title="Open the time filter">
    Go to the **Audit Logs** dashboard. Click the time selection button (shows **Last 30 Days** by default).
  </Step>
  <Step title="Select a period">
    Choose a preset period from the left panel, or click a date on the calendar widget.
  </Step>
  <Step title="Apply">
    Click **Apply**.
  </Step>
</Steps>

## Export Audit Logs

Export account-level audit logs as a `.csv` file. The export includes: Date and Time, Event Name, Event Category, Description, User/App Name, and IP Address.

<Steps>
  <Step title="Export">
    Go to the **Audit Logs** dashboard. Click the **Export** icon next to the **Filter By** icon.
  </Step>
</Steps>

A success message confirms the download. The file is saved as `Account_Audit_Logs`.
