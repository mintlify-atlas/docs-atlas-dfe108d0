## Audit Logs

**Audit Logs** provides full visibility into every user action and system event in your account. Each log entry captures the event name, category, user, date and time, and a detailed description. Filter logs by time period, category, event type, or user to isolate specific activity. Export logs as a CSV file for compliance reporting or offline analysis.

## Workflow analytics

The **Workflow Analytics Dashboard** tracks performance of deployed workflows and their versions. It shows success and failure rates for workflow runs, average response time (ART) per request, and event-based node execution counts — scoped to a selected time period and workflow version. Use it to identify failure patterns, detect performance bottlenecks, and compare performance across versions.

## Model analytics

The **Model Analytics Dashboard** tracks performance for fine-tuned, open-source, and external models. It measures model latency, request success and failure rates, scaling and replica usage, and credit consumption for deployments and fine-tuning.

## Model traces

**Model Traces** provides run-level analysis of any model deployed or integrated in your account. Each trace record includes the request input, generated output, response time, deployment version, and source metadata. Use it to troubleshoot individual runs, investigate failures, and validate model behavior.

## Custom scripts

**Monitoring > Custom Scripts** provides run-level and log-level visibility into custom scripts deployed in your account. The **All Runs** tab shows execution status, response time, function name, and source for each run. The **Logs** tab shows log entries captured during execution.

## Best practices

- Regularly review audit logs, performance metrics, and model run analytics for security and compliance.
- Use filters to focus on specific events or users.
- Use time-based filters for targeted analysis across specific date ranges.
