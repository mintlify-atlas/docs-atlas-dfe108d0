Generative AI Tools lets you integrate large language models (LLMs) into AI for Service apps. You can use pre-built integrations, bring your own model, or deploy Kore.ai's fine-tuned XO GPT models — and apply them across Automation AI, Search AI, Agent AI, Contact Center AI, and Quality AI.

To access Generative AI Tools, go to **Product Switcher** > **Generative AI Tools**, or select the icon in the left-side menu.

## Integration options

| Option | Description |
|--------|-------------|
| **Pre-built integrations** | Connect to OpenAI, Azure OpenAI, Anthropic, or Amazon Bedrock using pre-configured prompt templates. |
| **Bring Your Own Model (BYO)** | Integrate externally hosted or self-hosted enterprise models. Works with the Platform's Auth Profiles module. |
| **Kore.ai XO GPT** | Fine-tuned LLMs built for enterprise conversational AI — evaluated for accuracy, safety, and production readiness. |

## Key features

| Feature | Description |
|---------|-------------|
| **Models Library** | Connect to any LLM via pre-built integrations, custom integrations, or XO GPT. |
| **Prompts Library** | Create fully customized prompts optimized for your use case and model. |
| **Automation AI** | Auto dialog generation, utterance suggestions, conversation summaries, and more. |
| **Search AI** | AI-powered enterprise search and answer generation. |
| **Agent AI** | GenAI features for agent-facing workflows. |
| **Contact Center AI** | GenAI features for contact center operations. |
| **Quality AI** | AI-driven quality monitoring and analysis. |
| **Data Anonymization** | Mask PII and sensitive data before it reaches the LLM. |
| **Guardrails** | Enforce safety and appropriateness of LLM responses. |

## Free tokens

New apps in Standard workspaces receive free LLM tokens powered by OpenAI GPT-4o (Azure) via Kore.ai's enterprise account.

<Note>Free tokens are only available for apps created in Standard workspaces. Do not upload sensitive documents while using free tokens.</Note>

To claim free tokens, select the free tokens icon in the platform header.

## Getting started

<Steps>
  <Step title="Integrate an LLM">
    Integrate a [pre-built, custom LLM, or XO GPT model](/ai-for-service/generative-ai-tools/llm-integration) in Models Library.
  </Step>
  <Step title="Create prompts">
    Create [prompts](/ai-for-service/generative-ai-tools/prompts-library) in the Prompts Library.
  </Step>
  <Step title="Enable GenAI features">
    Enable GenAI Features for each product module.
  </Step>
  <Step title="Configure safeguards (optional)">
    Enable [Data Anonymization](/ai-for-service/generative-ai-tools/data-anonymization) and [Guardrails](/ai-for-service/generative-ai-tools/guardrails).
  </Step>
</Steps>

## Important considerations

| Topic | Details |
|-------|----------|
| **Language support** | LLMs generate responses in any app language that the LLM itself supports. |
| **Data sharing** | GenAI features send data to third-party LLM providers. Review your provider's data handling policies before enabling features in production. |
| **Free token security** | Do not upload sensitive documents while on free tokens. Switch to your own LLM account before handling confidential data. |
