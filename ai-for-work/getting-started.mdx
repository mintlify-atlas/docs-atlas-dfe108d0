## Platform access

Users log in to AI for Work at [https://work.kore.ai/login/](https://work.kore.ai/login/).

Admins can configure [user enrollments](/ai-for-work/administration/user-management) to manage who can access the platform. Users can also sign in using:

- Google Workspace or Microsoft Office profiles
- Your organization's SSO after configuring [Single Sign-On](/ai-for-work/administration/security-and-control#single-sign-on-sso)

**Supported browsers**: Google Chrome, Microsoft Edge, Apple Safari, and Mozilla Firefox (latest versions).

## Setup guide

Set up AI for Work for your organization in four phases.

<Steps>
  <Step title="Foundation">
    Get your account and access controls ready.

    | Step | Description |
    |------|-------------|
    | 1. Sign up | Contact the [Sales team](https://kore.ai/contact-us/) to create a new AI for Work account. |
    | 2. Configure user enrollments | Set up user enrollments to manage account access and invitations. [Learn more →](/ai-for-work/administration/user-management#user-enrollment) |
    | 3. Configure SSO | Set up Single Sign-On for easy access using your identity provider. [Learn more →](/ai-for-work/administration/security-and-control#single-sign-on-sso) |
    | 4. Add service accounts | Add Google, Microsoft, or LDAP service accounts to get user profiles and distribution lists. [Learn more →](/ai-for-work/administration/security-and-control#service-accounts) |
  </Step>
  <Step title="Configuration">
    Configure models, rate limits, and enterprise knowledge.

    | Step | Description |
    |------|-------------|
    | 1. Configure LLM and embedding models | Integrate with a pre-built or custom LLM. Pre-built integrations include Azure OpenAI, OpenAI, and Anthropic. Use the bring-your-own (BYO) framework for external or self-hosted models. [Learn more →](/ai-for-work/llm-configuration) |
    | 2. Configure rate limits | Set query rate limits per user category. Simple queries use 1 point; advanced queries use 3 points. Configure limits separately for Moderate Users and Power Users. [Learn more →](/ai-for-work/assistant-configurations#rate-limits) |
    | 3. Configure enterprise knowledge | Set up the Enterprise Knowledge Search AI app — a pre-configured fallback that generates responses when no specific agent matches the user's intent. [Learn more →](/ai-for-work/custom-agents/enterprise-knowledge) |
  </Step>
  <Step title="Deployment">
    Publish and configure agents for your users.

    | Step | Description |
    |------|-------------|
    | 1. Publish pre-built agents | Publish pre-built agents (content summarization, email drafting, translation, and more) from the Agent Store with minimal setup. [Learn more →](/ai-for-work/pre-built-agents) |
    | 2. Configure custom agents and applications | Build agents and applications that collect user input, retrieve data, connect to APIs, and execute complex tasks. [Learn more →](/ai-for-work/custom-agents) |
  </Step>
  <Step title="Go live">
    Invite your users to the platform.

    | Step | Description |
    |------|-------------|
    | 1. Invite users | Share access with your team. The AI for Work Assistant gives employees a single entry point to knowledge across systems and all published AI agents. [Learn more →](/ai-for-work/administration/user-management#user-enrollment) |
    | 2. Help users get started | Share guidance on how to use the AI for Work Assistant. [Learn more →](/ai-for-work/getting-started/how-to-use) |
  </Step>
</Steps>
