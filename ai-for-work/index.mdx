AI for Work addresses three common enterprise challenges: information silos that prevent employees from finding data across systems, repetitive manual tasks that drain productivity, and AI tools that historically required technical expertise to use.

## Key components

| Component | Description |
|---|---|
| **Access Points** | Users access the platform through web-based or mobile interfaces. |
| **Universal Orchestrator** | The central intelligence layer that coordinates interactions between all platform components. Maintains contextual awareness, routes requests to appropriate AI agents, executes multi-step flows, and manages handoffs between AI capabilities. |
| **AI Agents** | Includes Pre-built Solutions for common department needs (IT support, HR services), Pre-built Agents that can be customized with minimal setup, and Custom Agents built to meet specific enterprise requirements. |
| **Enterprise RAG Engine** | Powers intelligent information retrieval across structured and unstructured sources. Generates context-aware responses while maintaining security compliance. |
| **Security and Compliance** | Ensures adherence to enterprise security standards and regulatory requirements. |
| **Development Tools** | No-code builders, integration connectors, and built-in analytics for creating and optimizing AI solutions. |
| **Enterprise Systems** | Integrates with SharePoint, Salesforce, ServiceNow, enterprise databases, and email. |

## Agent types

| Agent type | Description | Use case |
|------------|-------------|----------|
| **Prompt Agent** | LLM-powered responses with custom instructions | General Q&A, content generation |
| **API Agent** | Connect to external APIs and services | Data retrieval, system integration |
| **Search Agent** | RAG-based retrieval from knowledge sources | Enterprise search, documentation |
| **Agentic Flow Agent** | Multi-step workflows with decision logic | Complex processes, approvals |
| **Autonomous Agent** | Self-directed task completion | Research, analysis, automation |
| **Bot Agent** | Conversational dialog with structured flows | Guided interactions, forms |
| **Workflow Agent** | Visual workflow automation | Business process automation |
| **MCP Agent** | Model Context Protocol integrations | External tool connections |

## Deployment channels

Deploy AI for Work agents across multiple channels:

- **Microsoft Teams** — Native Teams app integration
- **Slack** — Slack workspace deployment
- **Web Widget** — Embed in internal portals
- **AI for Work Assistant** — Standalone assistant with scheduling

## Implementation approach

The platform follows a phased deployment strategy. You start with core capabilities in the first few weeks, then progressively expand to custom solutions over the following months. Its modular structure lets you begin with pre-built solutions and extend with custom agents and integrations as your needs evolve.
